# Button

Button is an interactive button component with variants, sizes, color schemes, and loading states.

<StorybookLink storyId="components-button--basic">
  Open the full Button stories in Storybook
</StorybookLink>

## Quick Start

Import the component:

```tsx
import { Button } from "@tosui/react";
```

Use Button for the primary action in a view or form:

```tsx
<Button>Click me</Button>
```

## Common Patterns

### Variants

Use `variant` to change the visual weight of the action.

```tsx
<HStack gap={4}>
  <Button variant="solid">Save</Button>
  <Button variant="outline">Preview</Button>
  <Button variant="ghost">Cancel</Button>

```

Use `disabled` and `loading` to communicate unavailable or in-progress actions.

### Common Composition

Use Button with icons, as links, or as form actions. These patterns stay simple in docs and are easier to explore interactively in Storybook as the story set grows.

```tsx
import { ArrowRightIcon, DownloadIcon } from "your-icon-library";

<HStack gap={4}>
  <Button leftIcon={<DownloadIcon />}>Download</Button>
  <Button rightIcon={<ArrowRightIcon />}>Continue</Button>
  <Button leftIcon={<DownloadIcon />} rightIcon={<ArrowRightIcon />}>
    Both Icons
  </Button>
</HStack>
```

When loading, the left icon is replaced with a spinner:

```tsx
<Button leftIcon={<DownloadIcon />} loading>
  Downloading
</Button>
```

```tsx
<Button fullWidth>Continue</Button>
```

Button can render as any element using the `as` prop.

```tsx
<Button as="a" href="/signup">
  Sign Up
</Button>
```

For common form actions:

```tsx
<HStack gap={4} justify="flex-end">
  <Button variant="ghost">Cancel</Button>
  <Button>Submit</Button>
</HStack>
```

For destructive actions:

```tsx
<HStack gap={4}>
  <Button variant="outline">Cancel</Button>
  <Button colorScheme="error">Delete</Button>
</HStack>
```

For button groups:

```tsx
<HStack gap={0}>
  <Button rounded="md" roundedRight="none" borderRight="none">Left</Button>
  <Button rounded="none" borderRight="none">Middle</Button>
  <Button rounded="md" roundedLeft="none">Right</Button>
</HStack>
```

## API

### Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | `"solid"` \| `"outline"` \| `"ghost"` | `"solid"` | Visual variant |
| size | `"sm"` \| `"md"` \| `"lg"` | `"md"` | Button size |
| colorScheme | `"primary"` \| `"accent"` \| `"success"` \| `"warning"` \| `"error"` \| `"info"` | `"primary"` | Color scheme |
| disabled | `boolean` | `false` | Disable the button |
| loading | `boolean` | `false` | Show loading spinner |
| fullWidth | `boolean` | `false` | Expand to full width |
| leftIcon | `ReactNode` | - | Icon before text |
| rightIcon | `ReactNode` | - | Icon after text |
| as | `ElementType` | `"button"` | HTML element to render |

Button also accepts all [Box](/docs/components/box) styling props.

## Accessibility and Behavior

- Use `disabled` for unavailable actions.
- Use `loading` for in-progress actions; Tosui also disables the button while loading.
- If you render Button as a link with `as="a"`, ensure the destination and disabled behavior are both handled intentionally.

## Styling and Customization

- Use `variant`, `size`, and `colorScheme` for the built-in API.
- Use inherited Box props like `mt`, `w`, `rounded`, `px`, and responsive values for layout and visual adjustments.
- Reach for Storybook when validating combined states like `variant + colorScheme + size`.
